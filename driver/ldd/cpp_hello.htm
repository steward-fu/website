hello, world!永遠都是作為第一個教學範例，除了可以知道基本程式的框架之外，最主要的原因在於熟悉如何編譯和執行程式，司徒將以這樣的流程來介紹如何撰寫一個hello, world!驅動程式，而在此涉獵的Linux指令則必須依賴使用者先行上網學習，這部份是事前必須準備的東西，接著就先架設編譯環境。

Debian 9 (x64) 

$ cd 
$ wget https://github.com/steward-fu/pocketbeagle/releases/download/v1.0/gcc-4.9.tar.gz
$ tar xvf gcc-4.9.tar.gz
$ sudo mv gcc-4.9 /opt/
$ export PATH=$PATH:/opt/gcc-4.9/bin

$ wget https://github.com/steward-fu/pocketbeagle/releases/download/v1.0/kernel-4.1.18.tar.gz
$ tar xvf kernel-4.1.18.tar.gz
$ cd kernel
$ ARCH=arm CROSS_COMPILE=arm-linux-gnueabihf- make am335x_b4_deconfig
$ ARCH=arm CROSS_COMPILE=arm-linux-gnueabihf- make clean
$ ARCH=arm CROSS_COMPILE=arm-linux-gnueabihf- make zImage dtbs modules -j4
$ cd

main.c 

#include <linux/init.h>
#include <linux/device.h>
#include <linux/module.h>
 
MODULE_LICENSE("GPL");
MODULE_AUTHOR("Steward_Fu");
MODULE_DESCRIPTION("Linux Driver");
 
static int __init ldd_init(void)
{
  printk("hello, world!\n");
  return 0;
}
 
static void __exit ldd_exit(void)
{
  printk("unload it\n");
}
 
module_init(ldd_init);
module_exit(ldd_exit);

ldd_init()掛載驅動程式時，系統執行呼叫的副程式，目前僅列印訊息，其餘事情不處理。
ldd_exit()卸載驅動程式時，系統呼叫的副程式，目前僅列印訊息，其餘事情不處理。

Makefile 

export ARCH=arm
export CROSS_COMPILE=arm-linux-gnueabihf-
KERNEL=$(HOME)/kernel

obj-m += main.o
all:
  make -C $(KERNEL) M=$(PWD) modules

clean:
  make -C $(KERNEL) M=$(PWD) clean

接著輸入make指令編譯就可以產生main.ko

把main.ko複製到第一個FAT磁區，接著插入SDCard，開機執行如下指令即可： 

# cd /boot
# ls
MLO             main.ko         zImage
am335x-evm.dtb  u-boot.img

# insmod main.ko
[   15.059811] hello, world!

如果要卸載驅動程式，只要輸入如下指令即可 

# rmmod main
[   21.139729] unload it
